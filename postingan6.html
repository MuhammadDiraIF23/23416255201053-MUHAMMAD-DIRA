Apa Itu SQL?

SQL (Structured Query Language) adalah bahasa yang digunakan untuk berkomunikasi dengan database. SQL memungkinkan pengguna untuk melakukan berbagai operasi pada data dalam database, seperti mengambil, menyimpan, memperbarui, dan menghapus data. SQL digunakan dalam Sistem Manajemen Basis Data (DBMS) untuk mengelola informasi.

1. DDL (Data Definition Language)

DDL adalah bagian dari SQL yang digunakan untuk mendefinisikan struktur atau skema dari database, termasuk membuat, mengubah, dan menghapus tabel atau objek lainnya di dalam database.

Perintah-perintah DDL:

  • CREATE: Digunakan untuk membuat tabel baru.
  • ALTER: Digunakan untuk mengubah struktur tabel yang ada.
  • DROP: Digunakan untuk menghapus tabel.
  • TRUNCATE: Menghapus semua data dalam tabel, namun struktur tabel tetap ada.

Contoh DDL:

CREATE TABLE Pelanggan (
    ID INT PRIMARY KEY,
    Nama VARCHAR(100),
    Alamat VARCHAR(200)
);

2. DML (Data Manipulation Language)

DML adalah bagian dari SQL yang digunakan untuk manipulasi data dalam tabel. Dengan DML, kita bisa menambah, mengubah, dan menghapus data di dalam tabel yang sudah ada.

Perintah-perintah DML:

  • INSERT: Digunakan untuk menambahkan data ke dalam tabel.
  • UPDATE: Digunakan untuk memperbarui data yang sudah ada.
  • DELETE: Digunakan untuk menghapus data dari tabel.

Contoh DML:

INSERT INTO Pelanggan (ID, Nama, Alamat) 
VALUES (1, 'Ali', 'Jalan A');

UPDATE Pelanggan 
SET Alamat = 'Jalan B'
WHERE ID = 1;

DELETE FROM Pelanggan 
WHERE ID = 1;

3. DCL (Data Control Language)

DCL adalah bagian dari SQL yang digunakan untuk mengatur hak akses dan kontrol terhadap data dalam database.

Perintah-perintah DCL:

  • GRANT: Memberikan hak akses kepada pengguna.
  • REVOKE: Mencabut hak akses yang telah diberikan kepada pengguna.

Contoh DCL:

GRANT SELECT, INSERT ON Pelanggan TO user1;

REVOKE INSERT ON Pelanggan FROM user1;

4. TCL (Transaction Control Language)

TCL adalah bagian dari SQL yang digunakan untuk mengelola transaksi dalam database. Transaksi adalah serangkaian operasi yang dilakukan sebagai satu unit kerja yang tak terpisahkan.

Perintah-perintah TCL:

  • COMMIT: Menyimpan perubahan dalam transaksi.
  • ROLLBACK: Membatalkan perubahan dalam transaksi.
  • SAVEPOINT: Menetapkan titik pemulihan dalam transaksi.

Contoh TCL:

BEGIN TRANSACTION;

INSERT INTO Pelanggan (ID, Nama, Alamat) 
VALUES (2, 'Budi', 'Jalan C');

SAVEPOINT after_insert;

UPDATE Pelanggan
SET Alamat = 'Jalan D'
WHERE ID = 2;

ROLLBACK TO after_insert;

COMMIT;

Perbedaan Antara DDL, DML, DCL, dan TCL

Kategori Fungsi Utama Contoh Perintah Tujuan
DDL Mendefinisikan struktur database CREATE, ALTER, DROP, TRUNCATE Mengatur struktur database
DML Memanipulasi data dalam tabel INSERT, UPDATE, DELETE Menambah, memperbarui, dan menghapus data
DCL Mengelola hak akses pengguna GRANT, REVOKE Memberikan atau mencabut hak akses
TCL Mengelola transaksi dalam database COMMIT, ROLLBACK, SAVEPOINT Mengatur transaksi dan memastikan konsistensi
